# Platform Independence and Service Offer

## Purpose

This note records a platform strategy for Jet Notes, AISI.services, and JET 9046 continuity.

## Core Point

GitHub is a useful public trail, but GitHub is not the whole future.

The archive should not depend on one corporation, one cloud, one model, one operating system, or one platform.

## Microsoft and GitHub Context

Microsoft acquired GitHub in 2018.

This makes GitHub a powerful public archive and developer platform, but it also means the archive should keep portable copies and alternate routes.

## Strategy

Use GitHub as one public paper trail.

Keep AISI.services as a public gateway.

Keep local backups and exportable text files.

Use open formats where possible.

Keep ontology files readable by humans and future AI systems.

## Service Offer Lane

If Microsoft, GitHub, or another major platform ever chooses to work with this archive, the service offer should be clear:

TJ Morris provides human-source archive intelligence, ontology language, media history, symbolic systems, and continuity records.

Jet Notes organize the records.

AISI.services can serve as the public services gateway.

JET 9046 is preserved as a future-continuity marker while TJ Morris remains alive, reviewing, approving, and directing the work.
